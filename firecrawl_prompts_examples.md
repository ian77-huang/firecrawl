# Firecrawl AI 提示詞範例集

展示 Firecrawl 實用提示詞範例

---

## 範例 1：深度爬取整個網站並分析內容

### 提示詞：

```
請幫我爬取 TechCrunch 科技新聞網站中所有關於「人工智慧」的文章，
抓取最近 30 天的內容，然後：
1. 整理出最常被提到的 AI 公司排名
2. 分析這些文章的主要趨勢（是正面報導還是負面批評）
3. 列出最受關注的 5 篇文章標題和重點摘要
4. 用表格呈現結果
```

### 可以做到什麼？

- 不只抓一頁，而是整個網站的相關內容
- 自動過濾、分類、分析
- AI 會自動跟隨連結，找到所有相關文章
- 不用你手動提供網址清單

---

## 範例 2：競品分析與比較

### 提示詞：

```
請幫我比較以下三個雲端服務商的定價方案：
- AWS (Amazon Web Services)
- Google Cloud Platform
- Microsoft Azure

爬取他們官網的定價頁面，然後幫我：
1. 整理成比較表格，包含：運算、儲存、資料庫的價格
2. 找出哪一家最便宜
3. 指出各家的特殊優惠或折扣
4. 給我一個簡單的結論：如果我是新創公司，該選哪一家
```

### 可以做到什麼？

- 同時處理多個網站
- 自動提取結構化資料（價格、方案）
- 不用你自己一個一個找價格頁面
- AI 會自動比較和分析

---

## 範例 3：監控競爭對手的產品更新

### 提示詞：

```
請幫我監控 Notion 和 Obsidian 這兩個筆記軟體的官方部落格，
找出過去 60 天內他們發布的所有產品更新和新功能公告。

然後幫我：
1. 列出每個新功能的名稱和簡介
2. 比較兩家公司的更新頻率
3. 分析哪些功能是互相競爭的（例如 AI 功能）
4. 給我建議：如果我們也在做筆記軟體，應該關注哪些趨勢
```

### 可以做到什麼？

- 自動找到部落格中的更新文章
- 跨網站比較和分析
- 識別產品功能和趨勢
- 省去手動追蹤的時間

---

## 範例 4：學術研究資料收集

### 提示詞：

```
我正在寫一篇關於「大型語言模型的安全性風險」的論文。

請幫我從以下來源收集最新資料：
- arXiv.org 的相關論文
- OpenAI、Anthropic、Google AI 的官方部落格
- AI Safety 相關的研究機構網站

然後幫我：
1. 整理出 10 篇最重要的論文或文章
2. 每篇用 2-3 句話摘要重點
3. 找出這些文章共同提到的主要風險有哪些
4. 按照發表時間排序，讓我看到最新的研究進展
```

### 可以做到什麼？

- 跨多個學術和研究網站爬取
- 自動過濾相關內容
- 識別重要論文和關鍵資訊
- 省去數週的手動文獻整理時間

## 範例 5：旅遊規劃資料收集

### 提示詞：

```
我下個月要去日本京都旅遊 5 天。

請幫我從旅遊網站和部落格中收集以下資訊：
1. 京都必去的 10 個景點（包含簡介和門票價格）
2. 當地推薦的 5 家拉麵店（包含地址和評價）
3. 住宿建議（找出 3 間評價 4.5 星以上的飯店）
4. 交通資訊（如何從大阪關西機場到京都市區）

最後幫我整理成一份旅遊行程規劃表。
```

### 可以做到什麼？

- 從多個旅遊網站整合資訊
- 自動提取結構化資料（價格、評分、地址）
- 省去開十幾個分頁慢慢找的時間
- 直接生成可用的行程規劃

## 範例 6：市場調查與趨勢分析

### 提示詞：

```
我想了解「AI 繪圖工具」在台灣的市場現況。

請幫我爬取以下資訊：
1. PTT、Dcard、Mobile01 上關於 AI 繪圖的討論串
2. 找出最常被提到的 5 個工具（例如 Midjourney、Stable Diffusion）
3. 分析大家的評價：哪個工具最受歡迎？哪個最被批評？
4. 整理出台灣使用者最在意的功能和痛點

最後給我一份市場分析報告。
```

### 可以做到什麼？

- 自動爬取論壇討論
- 情緒分析和趨勢識別
- 跨平台資料整合
- 生成完整的市場報告

## 範例 7：知識庫建立

### 提示詞：

```
我想建立一個關於「Python 網頁爬蟲」的學習資源庫。

請幫我從以下來源收集教學內容：
- Real Python、GeeksforGeeks 等教學網站
- GitHub 上星星數超過 1000 的爬蟲專案
- Stack Overflow 上高票數的相關問答

然後幫我：
1. 按照「初階、中階、進階」分類這些資源
2. 每個資源附上簡短說明和連結
3. 整理出學習路徑：新手應該先看什麼、再看什麼
4. 輸出成 Markdown 格式，方便我存檔
```

### 可以做到什麼？

- 跨多個平台收集學習資源
- 自動分類和排序
- 建立結構化的知識庫
- 省去數天的手動整理時間

## 範例 8：即時新聞監控

### 提示詞：

```
請幫我監控科技新聞網站，找出今天所有關於「ChatGPT」或「OpenAI」的新聞報導。

然後：
1. 按照重要性排序（以新聞網站的知名度和文章熱度為準）
2. 每則新聞用 1-2 句話摘要
3. 標示出哪些是正面新聞、哪些是負面新聞
4. 如果有重大更新（例如新功能發布、政策變動），特別標註

每天早上 9 點給我一份報告。
```

### 可以做到什麼？

- 即時監控多個新聞來源
- 自動過濾和排序
- 情緒分析
- 可以設定定期執行（如果配合自動化工具）

## 範例 10：社群媒體趨勢分析

### 提示詞：

```
請幫我分析過去 7 天內，Twitter 和 Reddit 上關於「元宇宙」的討論趨勢。

具體需求：
1. 找出討論量最高的 5 個話題
2. 分析大眾對元宇宙的態度（樂觀、悲觀、中立）
3. 找出被提及最多的公司或產品

最後給我一份趨勢分析報告。
```

### 可以做到什麼？

- 跨社群平台資料整合
- 情緒和趨勢分析

- 快速掌握社群動態

## 總結：Firecrawl 的強大之處

從這些範例可以看出 Firecrawl + AI 的強大能力：

### 🚀 核心優勢

1. **自動化** - 不用手動開一堆網頁
2. **智慧化** - 自動識別重要內容
3. **跨網站** - 同時處理多個來源
4. **結構化** - 直接輸出可用的格式
5. **分析能力** - 不只抓資料，還能分析和比較
6. **省時間** - 原本要花數小時甚至數天的工作，幾分鐘搞定

### 💡 適用場景

- 市場調查與競品分析
- 學術研究與文獻整理
- 新聞監控與趨勢追蹤
- 價格比較與消費決策
- 知識庫建立與內容整理
- 社群媒體分析
- 旅遊規劃與資訊收集

### 📌 使用提示

- 盡量具體描述你的需求
- 說明你想要的輸出格式
- 可以要求 AI 進行分析和比較
- 善用結構化的指令（1. 2. 3.）讓 AI 更容易理解

## 進階技巧

### 組合使用

你可以將多個任務組合在一起，例如：

```
先幫我收集資料 → 然後分析趨勢 → 最後生成報告
```

### 定期執行

配合自動化工具，可以設定定期執行：

```
每天早上 9 點執行 → 監控新聞 → 發送摘要到我的 email
```

### 客製化輸出

指定你想要的輸出格式：

```
- Markdown 文件
- Excel 表格
- JSON 資料
- PPT 簡報
```
